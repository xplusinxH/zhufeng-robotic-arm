# Python 中文注释规范

本文档约定本项目后续 Python 代码的注释标准。目标是让 Jetson 现场调试、视觉标定、串口联调和后续交接时都能快速读懂代码意图，而不是堆砌低价值注释。

## 基本原则

1. 模块顶部必须写中文模块说明，说明职责、输入输出、运行环境和边界。
2. 公共函数、公共类必须写中文 docstring，说明参数含义、返回值、单位和异常条件。
3. 坐标系、单位换算、串口协议、硬件依赖、标定文件格式必须有注释。
4. 算法关键步骤必须注释原因，例如为什么取中位数、为什么深度对齐到彩色图、为什么缓存底座 tag。
5. 不写无意义注释，例如“给变量赋值”“循环列表”。
6. 注释与代码不一致时，优先修改注释；不允许保留过期注释。

## 推荐格式

模块说明：

```python
"""D435 相机采集封装。

本模块负责启动彩色流和深度流，并将深度帧对齐到彩色帧。
业务层不直接依赖 pyrealsense2，便于 PC 端测试和 Jetson 端资源释放。
"""
```

函数说明：

```python
def pixel_depth_to_camera(u, v, depth_m, intrinsics):
    """将对齐像素和深度转换为相机坐标。

    参数：
    - u, v：彩色图像素坐标。
    - depth_m：该像素处深度，单位米。
    - intrinsics：包含 fx/fy/cx/cy 的内参字典，单位像素。

    返回：
    - 有效深度时返回 (x, y, z)，单位米。
    - 深度无效时返回 None。
    """
```

关键逻辑注释：

```python
# 深度帧已经对齐到彩色坐标系，因此这里用彩色内参做像素转三维。
camera_point = pixel_depth_to_camera(u, v, depth_m, intrinsics)
```

## 必须标明的项目约定

- 相机坐标系：D435 SDK 坐标系，单位默认米。
- 串口输出：旧目标协议坐标使用毫米，新 AprilTag JSON 使用米。
- 图像格式：OpenCV 使用 BGR，深度图原始值需乘以 depth_scale 才是米。
- 标定文件：内参、桌面平面、外参必须注明单位和坐标系。
- Jetson 依赖：涉及 `pyrealsense2`、`cv2`、`serial`、`pupil_apriltags` 的模块必须说明真机依赖。

## 不推荐

```python
# 设置 x
x = 1

# 遍历数组
for item in items:
    ...
```

这类注释没有解释业务意图，后续不要新增。
