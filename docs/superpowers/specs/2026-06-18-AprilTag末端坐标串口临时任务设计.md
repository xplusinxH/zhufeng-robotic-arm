# AprilTag 末端 6D 位姿串口任务设计

## 目标

在 Jetson + RealSense D435 视觉侧实现一个临时标定数据输出程序。Jetson 只负责视觉测量，不控制机械臂，不接入 CAN，不写电机参数。

视觉侧职责固定为三件事：

1. 从 D435 获取 RGB 图像。
2. 检测 `tag36h11` 中的底座参考 tag 和末端 tag。
3. 输出 `tag_base_ref -> tag_tool0` 的 6D 位姿。

本任务输出给电脑端用于后续联合优化。因为后续需要求解 `tool0 -> tag_tool0`、腕部关节零偏和末端姿态误差，所以不能只输出 xyz 坐标，必须输出完整 6D 位姿。

## Tag 与坐标约定

本轮已确认采用以下 ID 约定：

```text
tag_base_ref = ID 1
tag_tool0    = ID 0
```

坐标系：

- `C`：D435 彩色相机坐标系。
- `B`：`tag_base_ref` 坐标系。
- `T`：`tag_tool0` 坐标系。

每帧 AprilTag 检测得到：

```text
T_C_B = camera -> tag_base_ref
T_C_T = camera -> tag_tool0
```

需要输出：

```text
T_B_T = inverse(T_C_B) * T_C_T
```

也就是 `tag_tool0` 在 `tag_base_ref` 坐标系下的 6D 位姿。

## 数据采样策略

单帧 AprilTag 位姿会抖动，因此 Jetson 不应直接发送单帧结果。每个 sample 默认连续采集 15 帧，可配置为 10 到 20 帧。

每个 sample 中：

- 只使用同时看到 `tag_base_ref` 和 `tag_tool0` 的有效帧。
- 位置部分使用逐轴中位数。
- 姿态部分先将四元数对齐到同一半球，再做平均并归一化。
- 如果有效帧数不足，则输出无效质量状态，不发送伪造位姿。

## 输出协议

串口主输出改为 newline-delimited JSON。每条 JSON 独占一行，方便电脑端串口程序按行解析。

推荐字段：

```json
{
  "protocol": "sukinee_tag_pose_v1",
  "sample_id": "S0001",
  "seq": 1,
  "timestamp_jetson": "2026-06-18T12:00:00.000+08:00",
  "from_frame": "tag_base_ref",
  "to_frame": "tag_tool0",
  "tag_family": "tag36h11",
  "tag_base_ref_id": 1,
  "tag_tool0_id": 0,
  "tag_size_m": 0.08,
  "position_m": [0.0, 0.0, 0.0],
  "orientation_xyzw": [0.0, 0.0, 0.0, 1.0],
  "frame_count_used": 15,
  "quality": {
    "both_tags_seen": true,
    "base_ref_seen": true,
    "tool0_seen": true,
    "mean_reprojection_error_px": null,
    "decision_margin_min": null,
    "hamming_max": 0
  },
  "crc32": null
}
```

`crc32` 当前保留为 `null`，后续如果电脑端要求再启用。

## 串口触发方式

第一版采用“收到查询后采样并返回”的方式，避免 Jetson 持续刷屏影响电脑端调试：

```text
电脑端发送：@GET_TAG_POSE#
Jetson 返回：一行 sukinee_tag_pose_v1 JSON
```

如果后续需要高帧率连续输出，可以增加 `--stream` 模式，让 Jetson 按固定频率持续发送 sample JSON。

旧的 `@GET_TOOL#` / `@TOOL,x,y,z#` 三坐标协议只作为历史调试协议，不作为本轮标定主协议。

## 错误与质量状态

如果 sample 中有效帧不足，Jetson 仍返回 JSON，但质量字段必须明确标出失败原因：

- `both_tags_seen = false`
- `base_ref_seen` 表示是否至少看见过底座参考 tag。
- `tool0_seen` 表示是否至少看见过末端 tag。
- `frame_count_used` 为实际用于融合的有效帧数。
- `position_m` 和 `orientation_xyzw` 为 `null`。

这样电脑端可以保存失败样本，而不是把串口超时误判为程序崩溃。

## 软件模块

需要升级或新增：

- `vision/apriltag/pose_service.py`：从检测结果计算 `tag_base_ref -> tag_tool0` 的 6D 位姿。
- `vision/apriltag/pose_sample.py`：对 10 到 20 帧位姿做稳健融合。
- `vision/coordinate/pose_transform.py`：增加旋转矩阵到四元数、四元数平均和位姿提取。
- `vision/communication/tag_pose_protocol.py`：输出 `sukinee_tag_pose_v1` JSON。
- `vision/tools/serve_tag1_pose_serial.py`：改为按查询采样并返回 JSON。
- `vision/config.yaml` 和 `vision_jeason/config.yaml`：默认 tag ID 改为 `base=1`、`tool=0`，tag 尺寸为 `80 mm`。

## 验收标准

1. PC 端单元测试覆盖相对位姿、四元数输出、sample 融合和 JSON 协议。
2. `vision_jeason` 中同步包含 Jetson 运行所需文件。
3. Jetson 上能识别 ID1 与 ID0，并在串口收到 `@GET_TAG_POSE#` 后返回一行 JSON。
4. 返回 JSON 中 `from_frame` 为 `tag_base_ref`，`to_frame` 为 `tag_tool0`。
5. 返回 JSON 中包含 `position_m` 和 `orientation_xyzw`，而不是只有 xyz。
6. 任一 tag 不可见或有效帧不足时，返回失败质量状态 JSON。
