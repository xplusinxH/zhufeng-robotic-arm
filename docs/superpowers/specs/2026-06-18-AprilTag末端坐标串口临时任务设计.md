# AprilTag 末端坐标串口临时任务设计

## 目标

在现有 Jetson + D435 视觉系统上增加一个临时测量程序：

```text
识别底座 tag0
识别末端 tag1
计算 tag1 相对 tag0 的三维坐标
收到串口查询时返回当前末端坐标
```

本任务用于临时联调，不执行机械臂控制，也不替代后续正式联合优化标定。

## 坐标约定

本设计采用用户本次指定的 tag 编号：

- `tag0`：固定在机械臂底座或底座附近的静止参考 tag。
- `tag1`：固定在执行器末端刚性结构上的末端 tag。

坐标系：

- `C`：相机彩色光学坐标系。
- `B0`：底座 `tag0` 坐标系。
- `T1`：末端 `tag1` 坐标系。

相机每帧检测得到：

```text
T_C_B0 = camera -> tag0
T_C_T1 = camera -> tag1
```

末端 tag 在底座 tag 坐标系下的位姿为：

```text
T_B0_T1 = inverse(T_C_B0) * T_C_T1
```

临时串口返回的坐标默认使用 `T_B0_T1` 的平移部分，也就是：

```text
tag1 原点在 tag0 坐标系下的位置
```

如果后续需要返回真实工具中心点，需要额外提供 `tag1 -> tool_center` 的固定偏移。

## 推荐方案

### 方案 A：直接返回 tag1 原点坐标，推荐先做

Jetson 同时检测 tag0 和 tag1，实时维护最近一次有效的 `tag1 in tag0` 坐标。
对方串口查询时，Jetson 返回该坐标。

优点：

- 实现最快。
- 不需要机械臂运动学。
- 不需要手测复杂外参。
- 可直接验证视觉链路和串口链路。

限制：

- 返回的是 tag1 原点，不一定是真正工具中心点。
- tag1 安装位置会影响坐标含义。

### 方案 B：返回工具中心点坐标

在方案 A 基础上，增加 `tag1 -> tool_center` 的固定偏移配置，返回工具中心点在 tag0
坐标系下的位置。

优点：

- 更接近实际控制需求。

限制：

- 需要准确测量或标定 tag1 到工具中心点的固定偏移。
- 比方案 A 多一个误差来源。

### 方案 C：接入正式联合优化标定

使用多姿态样本反解 `base_link -> camera`、`tool0 -> tag_tool0` 和关节零位偏差。

优点：

- 更正式、更完整。

限制：

- 采样和优化流程较长。
- 不适合作为当前“对方查询就返回坐标”的临时联调任务第一步。

本阶段推荐先采用方案 A。

## 软件结构

新增或修改模块：

- `vision/apriltag/`：AprilTag 检测与位姿估计。
- `vision/coordinate/pose_transform.py`：4x4 位姿矩阵求逆、相乘和坐标提取。
- `vision/communication/tag_pose_protocol.py`：串口查询与返回帧格式。
- `vision/tools/serve_tag1_pose_serial.py`：主程序，负责相机、tag 检测和串口服务。
- `vision/config.yaml`：增加 tag 尺寸、串口端口和输出单位配置。

## 运行流程

1. Jetson 启动 D435 彩色图采集。
2. 程序加载相机内参。
3. 每帧检测 AprilTag。
4. 若同时看到 tag0 和 tag1，计算 `T_B0_T1`。
5. 将最近一次有效坐标缓存在内存中，并记录时间戳。
6. 串口收到查询命令。
7. 如果最近坐标仍有效，返回坐标。
8. 如果 tag 缺失或数据超时，返回无效状态。

## 串口协议草案

查询命令建议：

```text
@GET_TOOL#
```

有效返回建议：

```text
@TOOL,x_mm,y_mm,z_mm,age_ms#
```

示例：

```text
@TOOL,123.4,-56.7,345.6,42#
```

无有效数据返回：

```text
@NO_TAG#
```

错误返回：

```text
@ERR,message#
```

最终协议必须与对方确认后再固化。

## 错误处理

- 看不到 tag0：不能建立底座参考，返回 `@NO_TAG#`。
- 看不到 tag1：没有末端坐标，返回 `@NO_TAG#`。
- 最近有效坐标超时：返回 `@NO_TAG#`。
- 串口无法打开：程序启动失败并打印端口。
- tag 尺寸未配置：程序启动失败。
- 检测到 tag id 与配置不一致：忽略该 tag。

## 验收标准

第一阶段验收：

1. 画面中同时出现 tag0 与 tag1 时，程序能稳定显示两者检测结果。
2. 移动末端 tag1 时，输出坐标随之变化。
3. 固定 tag1 时，坐标抖动在可接受范围内。
4. 串口收到查询命令后能返回当前坐标。
5. 任一 tag 不可见时，串口返回无有效 tag 状态。

## 待确认问题

1. 对方串口查询命令是否接受 `@GET_TOOL#`。
2. 返回坐标单位是否使用毫米。
3. 返回坐标是否只需要位置，还是还需要姿态。
4. `tag0` 和 `tag1` 的实际黑白图案边长。
5. `tag1` 返回点是否就是末端 tag 原点，还是需要换算到工具中心点。
