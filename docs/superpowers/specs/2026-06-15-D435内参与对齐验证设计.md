# D435 内参与 RGB-Depth 对齐验证设计

## 目标

读取并保存 D435 在当前运行分辨率下的相机内参，并通过 Jetson 屏幕实时验证
彩色图与深度图的对齐效果。用户能够点击彩色图像素，查看该点深度和相机坐标系
三维坐标，并保存验证数据。

本阶段只处理 D435 相机坐标系，不涉及机械臂工作坐标系转换。

## 运行条件

- Jetson Nano 连接显示器、键盘和鼠标。
- D435 通过 USB 3.x 连接。
- 彩色流和深度流使用 640x480、30 FPS。
- 项目运行目录为 `/mnt/zhufeng_data/zhufeng/vision_project`。
- 标定配置保存在 eMMC，调试数据保存在外部 ext4 存储。

## 方案

采用实时交互验证为主、保存验证数据为辅的方式。

程序在 Jetson 屏幕上显示：

- 左侧：RGB 彩色图。
- 右侧：对齐到彩色图后的深度热力图。
- 点击位置：十字标记、像素坐标、深度值和相机三维坐标。

交互方式：

- 鼠标左键：选择测量点。
- `S`：保存当前彩色图、原始深度图、深度热力图、叠加图和测量结果。
- `Q` 或 `Esc`：退出。

## 组件设计

### 相机采集封装

扩展 `camera/realsense_camera.py`，在相机启动后提供：

- 彩色相机内参。
- 对齐后的深度相机内参。
- 深度比例尺。
- 相机序列号和固件版本。
- 对齐后的彩色帧与深度帧。

所有接口保持 Python 3.6 兼容。

### 内参保存工具

实现 `calibration/calibrate_intrinsic.py`：

1. 启动 D435。
2. 读取当前 640x480 彩色流和对齐深度流的内参。
3. 读取深度比例尺、相机序列号和固件版本。
4. 将结果写入：

```text
/etc/zhufeng-vision/calibration/camera_intrinsic.json
```

文件使用 JSON 格式，并明确记录单位、分辨率和采集时间。

### 实时对齐验证工具

新增 `tools/check_alignment.py`，负责：

1. 采集对齐后的彩色帧和深度帧。
2. 将深度图转换为便于观察的热力图。
3. 并排显示彩色图与深度热力图。
4. 接收鼠标点击，在彩色图对应位置读取深度。
5. 使用内参将像素和深度转换为相机三维坐标。
6. 在画面中显示测量值。
7. 响应保存和退出按键。

## 数据流

```text
D435 彩色帧与深度帧
→ 深度对齐到彩色帧
→ 获取点击像素深度
→ 像素与深度转换为相机三维坐标
→ 屏幕显示与人工检查
→ 按需保存验证数据
```

## 文件存储

eMMC 保存最终内参：

```text
/etc/zhufeng-vision/calibration/camera_intrinsic.json
```

外部盘保存调试数据：

```text
/mnt/zhufeng_data/zhufeng/data/alignment/<时间戳>/
├── color.png
├── depth_raw.png
├── depth_colormap.png
├── overlay.png
└── measurement.json
```

原始深度图使用无损格式保存，保留测量值，不只保存热力图。

## 异常处理

- 相机启动失败时打印明确错误并退出。
- 彩色帧或深度帧无效时跳过当前帧，不保存错误数据。
- 点击位置深度为 0 或超出有效范围时显示“无有效深度”。
- eMMC 标定目录不存在时创建目录；权限不足时打印目标路径并退出。
- 外部盘未挂载或不可写时禁止保存调试数据，但实时查看仍可继续。
- 程序退出时始终停止相机管线并关闭 OpenCV 窗口。

## 测试策略

PC 端使用轻量测试验证：

- 内参字段序列化。
- 像素和深度转三维坐标。
- 无效深度处理。
- 保存目录和文件命名。
- Python 3.6 语法兼容。

PC 端不模拟实时画面、鼠标交互或完整 RealSense 硬件行为。

Jetson 真实硬件测试：

- 内参文件成功写入 eMMC。
- 实时窗口正常显示。
- 鼠标点击能够显示合理深度与三维坐标。
- 彩色图和深度热力图中的物体边缘基本重合。
- `S` 键能够保存完整验证数据。
- `Q` 和 `Esc` 能够正常释放相机并退出。

## 验收标准

- `camera_intrinsic.json` 包含分辨率、`fx`、`fy`、`cx`、`cy`、畸变参数、
  深度比例尺、序列号和固件版本。
- 点击彩色图中的物体时，显示的深度来自同一物体。
- 移动物体时，深度热力图区域与彩色物体同步移动。
- 物体主要边缘无明显大范围错位。
- 保存的数据能够用于后续复查和桌面平面标定准备。
