# D435 内参与 RGB-Depth 对齐验证实施计划

> **执行要求：** 使用 `superpowers:subagent-driven-development`（推荐）或
> `superpowers:executing-plans` 按任务逐项实施。步骤使用复选框跟踪。

**目标：** 读取并保存 D435 内参，在 Jetson 屏幕上实时验证 RGB 与深度对齐，并支持点击测量和保存验证数据。

**架构：** 相机封装负责提供对齐帧、内参与设备信息；纯函数模块负责内参序列化和像素转三维坐标；Jetson 工具负责 OpenCV 实时窗口、鼠标交互与数据保存。PC 端只测试纯函数和文件格式，真实画面与交互在 Jetson 真机验收。

**技术栈：** Python 3.6、OpenCV 3.2、NumPy、pyrealsense2、JSON、unittest。

---

### 任务 5：实现 Jetson 实时对齐验证工具

**文件：**

- 新建：`vision/tools/check_alignment.py`
- 使用：`vision/tools/alignment_data.py`

- [ ] **步骤 1：实现实时窗口**

窗口并排显示彩色图和对齐后的深度热力图。深度显示范围固定为 0.15 米到 1.20 米，
避免每帧颜色范围变化。

- [ ] **步骤 2：实现鼠标点击测量**

鼠标左键只响应左侧彩色图区域。点击后读取同位置对齐深度，计算并显示：

```text
像素坐标、深度米数、X/Y/Z 相机坐标
```

深度为 0 时显示“无有效深度”。

- [ ] **步骤 3：实现保存与退出**

- `S`：保存 `color.png`、`depth_raw.png`、`depth_colormap.png`、`overlay.png`
  和 `measurement.json`。
- `Q` 或 `Esc`：停止相机并关闭窗口。

默认保存根目录：

```text

```
